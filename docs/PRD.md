# SkyPulse · 实时航班动态 AI 解说大屏 — 产品需求文档(PRD)

> 版本 v0.2 · 状态:待确认 · 适用区域:可切换(默认京津冀)/ 跟随视野 / 全球 · 分辨率:1920×1080

---

## 1. 目标

构建一个**单页大屏**应用 SkyPulse:在深色地图上实时渲染**当前聚焦区域**(默认京津冀,可切换为其它预设区域、跟随地图视野或全球视图)的飞机,并由一个 **AI 解说员**周期性产出**中性、克制**的区域空域态势解说;在检测到紧急代码(7500/7600/7700)、快速下降、疑似盘旋等异动时,给出**客观说明其通常含义与管制处置**的解释。

定位是**学习/演示/可视化**,**非航行、非调度用途**。界面常驻免责声明。

成功标准(高层):
- 打开页面即可看到当前聚焦区域上空飞机按航向旋转、按高度着色、平滑滑行;可切换区域 / 跟随视野 / 切到全球视图,数据随之更新。
- 每约 30s 出现一段新的区域态势解说;异动发生时出现一条结构化告警卡片。
- OpenSky 凭证与 DeepSeek 密钥**从不**出现在前端产物或网络响应中。

---

## 2. 核心功能

1. **后端单点轮询 OpenSky**:用 OAuth2 client-credentials 取 access token,按**当前活动 bbox**调 `/api/states/all`(全球视图时不带 bbox),约 **10s** 一次;令牌到期前刷新;遇到 429/5xx 指数退避。
2. **多区域 / 全球视图**:支持三种取数范围模式——
   - **预设区域**:京津冀、长三角、粤港澳、华北等下拉切换;
   - **跟随视野(viewport)**:以当前地图可视范围为 bbox,停止拖动/缩放后防抖更新;
   - **全球**:缩放到阈值以下(或显式选择)时切到全球 `states/all`,以**更长间隔**轮询(见 §6 限流护栏)。
   切换通过前端→后端的控制通道下发,后端据此调整轮询 bbox(仍为单点轮询,见 §6)。
3. **SSE 推送**:后端通过 Server-Sent Events 把每个轮询周期的飞机快照推给前端;快照携带当前 `bbox`/`mode`/`region`(事件类型见 §4)。
4. **地图渲染(react-leaflet)**:深色底图,飞机图标按 `true_track` 旋转、按 `baro_altitude` 着色;点击飞机显示详情;顶栏显示当前区域/模式、数据时延、在视飞机数。
5. **客户端插值**:相邻两次快照之间按速度/航向对飞机位置做线性/航位推算插值(约 60fps),实现平滑滑行;新快照到达或**切换范围**后平滑校正(切换时清理不在新范围的目标)。
6. **AI 区域解说**:后端每约 **30s** 用当前快照的聚合特征(飞机数、进出港趋势、高度分布、热点区)与**当前范围(区域名/视野/全球)**调用 DeepSeek 生成 1–3 句中性解说,经 SSE 推前端解说流。
7. **异动检测与告警**:后端维护短时轨迹窗口,检测紧急代码 / 快速下降 / 疑似盘旋;命中后生成结构化告警 + 一段客观解释,经 SSE 推前端告警面板。
8. **合规提示常驻**:界面固定显示「数据可能延迟/不完整,仅供学习,不用于实际航行或调度」。

## 3. 明确不做(Out of Scope)

- ❌ 不做飞机出事 / 坠机 / 事故的判断、预测或渲染;告警**只**解释代码/现象的通常含义。
- ❌ 不做用户账户、登录、权限体系。
- ❌ 不做历史回放、数据落库长期存储(仅内存中的短时轨迹窗口)。
- ❌ 不做移动端 / 响应式适配(仅针对 1920×1080 大屏)。
- ❌ 全球视图**不**逐架精细呈现到底(可能聚合/抽稀,见 §6),也不保证全球数据完整或实时。
- ❌ 不在前端持有或代理任何密钥;前端**不**直连 OpenSky 或 DeepSeek。
- ❌ 不提供任何"建议规避/改航/处置"类操作性结论。
- ❌ 不做航司/机型富数据补全(v0.1 只用 OpenSky states 自带字段;callsign 直接展示)。

## 4. 数据来源与契约

### 4.1 上游:OpenSky `/api/states/all`

- 认证:OAuth2 client-credentials,token endpoint 取 access_token,`Authorization: Bearer`。
- 请求:按**当前活动 bbox**带 `lamin/lomin/lamax/lomax`(如 `lamin=38.0&lomin=114.0&lamax=41.5&lomax=119.0`);**全球模式**不带 bbox(`/states/all` 返回全量,数据量大、扣额度高)。
- 轮询间隔:区域/视野模式 **10s**(`POLL_INTERVAL_MS`,默认 10000);全球模式用更长间隔 **`GLOBAL_POLL_INTERVAL_MS`(默认 30000)**(见 §6 护栏)。
- 限流策略:token 过期前刷新;429/5xx → 指数退避(基数 2s,上限 60s,带抖动);连续失败时前端显示"数据暂不可用"。
- states 原始数组字段(索引)→ 后端归一化为对象:

| 字段 | 来源(states index) | 说明 |
|---|---|---|
| `icao24` | [0] | 唯一标识(主键) |
| `callsign` | [1] | 呼号(可能为空,trim) |
| `origin_country` | [2] | 注册国 |
| `time_position` | [3] | 最后位置时间(s) |
| `last_contact` | [4] | 最后联系时间(s) |
| `longitude` | [5] | 经度 |
| `latitude` | [6] | 纬度 |
| `baro_altitude` | [7] | 气压高度(m),用于着色 |
| `on_ground` | [8] | 是否在地面 |
| `velocity` | [9] | 地速(m/s) |
| `true_track` | [10] | 航向(°),用于旋转 |
| `vertical_rate` | [11] | 垂直速率(m/s),用于下降检测 |
| `geo_altitude` | [13] | 几何高度(m) |
| `squawk` | [14] | 应答机代码,用于紧急代码检测 |
| `spi` | [15] | 特殊用途指示 |
| `position_source` | [16] | 位置来源 |

> 缺失位置(lat/lon 为 null)或 `on_ground=true` 的目标:可选过滤;v0.1 默认渲染在空目标,地面目标灰显或过滤(实现切片中定)。

### 4.2 前端 → 后端 控制通道(切换取数范围)

端点 `POST /api/region`(JSON)。后端据此切换**单点轮询**的活动范围(全局共享,见 §6),并在下一周期生效:

```json
// 预设区域
{ "mode": "preset", "region": "yrd" }
// 跟随视野:传当前地图可视 bbox [lamin, lomin, lamax, lomax]
{ "mode": "viewport", "bbox": [29.0, 118.0, 33.0, 123.0] }
// 全球
{ "mode": "global" }
```

- `region` 取值:`bj`(京津冀,默认)、`yrd`(长三角)、`gba`(粤港澳)、`ncn`(华北)等,预设表在后端配置。
- `viewport` 的 bbox 由后端做**面积/纬度护栏**(见 §6):超限则拒绝或自动夹取并回传实际生效范围。
- 后端响应当前生效的 `{ mode, region?, bbox?, pollIntervalMs }`;随后通过 SSE `status`/`snapshot` 广播给所有连接(单点轮询,所有大屏看到同一活动范围)。

### 4.3 后端 → 前端 SSE 事件契约

SSE 端点 `GET /api/stream`。事件类型:

**事件 `snapshot`**(每轮询周期):
```json
{
  "type": "snapshot",
  "serverTime": 1730000000000,
  "dataAgeSec": 4,
  "count": 137,
  "mode": "preset | viewport | global",
  "region": "bj",
  "bbox": [38.0, 114.0, 41.5, 119.0],
  "aircraft": [
    {
      "icao24": "780abc",
      "callsign": "CCA1234",
      "lat": 39.9, "lon": 116.4,
      "baroAltitude": 9144,
      "geoAltitude": 9200,
      "velocity": 230.5,
      "track": 87.3,
      "verticalRate": -2.1,
      "onGround": false,
      "squawk": "2000",
      "lastContact": 1729999996
    }
  ]
}
```

**事件 `narration`**(每约 30s):
```json
{
  "type": "narration",
  "id": "n_1730000000",
  "ts": 1730000000000,
  "text": "当前京津冀空域约有 130 架在视航空器,进港流主要集中在首都机场以东……",
  "stats": { "count": 130, "climbing": 22, "descending": 18, "cruising": 90 }
}
```

**事件 `alert`**(命中异动时):
```json
{
  "type": "alert",
  "id": "a_780abc_7700_1730000000",
  "ts": 1730000000000,
  "category": "emergency_code | rapid_descent | suspected_holding",
  "severity": "info | notice",
  "icao24": "780abc",
  "callsign": "CCA1234",
  "evidence": {
    "squawk": "7700",
    "verticalRate": -12.4,
    "headingChangeDeg": 285,
    "windowSec": 90
  },
  "text": "该航空器应答机显示 7700(通用紧急)。7700 通常表示机组宣布紧急状态,管制会优先提供协助与隔离。以上为代码通常含义,不代表该航班发生事故。"
}
```

可选事件:`status`(轮询健康度 / 退避中 / 上游不可用)、`heartbeat`(保活)。

### 4.4 异动判定(标准阈值)

- **紧急代码**:`squawk ∈ {7500, 7600, 7700}` → 立即告警。文案分别说明:7500 通常表示非法干扰/劫持类,7600 通常表示通信失效,7700 通常表示通用紧急;均说明管制通常的优先协助/隔离处置。**绝不臆断具体原因或后果**。
- **快速下降**:`verticalRate ≤ -10 m/s`(≈ -2000 ft/min)且**持续 ≥ 2 个轮询周期**。
- **疑似盘旋**:在 **~90s** 窗口内**累计航向变化 ≥ 270°**,且地速较低 / 水平位移较小。
- 去抖动:同一 `icao24` 同 `category` 的告警在冷却期(默认 5 分钟)内不重复推送;状态恢复后可再次触发。

## 5. 技术栈与数据流

### 5.1 技术栈
- **前端**:Vite + React + TypeScript + Tailwind CSS + react-leaflet(Leaflet 深色瓦片)。
- **后端**:Node + Express + TypeScript;`openai` 包接 DeepSeek(`baseURL: https://api.deepseek.com`);SSE 原生实现。
- **AI 模型**:解说与告警均用 `deepseek-v4-flash`。
- **配置**:后端 `.env`(`OPENSKY_CLIENT_ID/SECRET`、`DEEPSEEK_API_KEY`、`POLL_INTERVAL_MS`、`GLOBAL_POLL_INTERVAL_MS`、`NARRATION_INTERVAL_MS`、默认 `REGION_BBOX`/`REGION_NAME`、视野护栏 `MAX_VIEWPORT_DEG`、阈值);前端仅需后端基址。

### 5.2 数据流
```
                       前端切换范围(POST /api/region: preset|viewport|global)
                                          │
                                          ▼  设置活动 bbox / 间隔
OpenSky /states/all ──(轮询, OAuth2+退避)──▶ 后端 Poller(单点,活动范围全局共享)
                                                     │ 归一化 + 维护轨迹窗口
                          ┌──────────────────────────┼───────────────────────────┐
                          ▼                          ▼                           ▼
                   每周期 snapshot           每30s 聚合→DeepSeek 解说       异动检测→DeepSeek 解释
                          │   (带 mode/region/bbox)   │                           │
                          └──────────── SSE /api/stream (snapshot|narration|alert|status) ┘
                                                     │
                                                     ▼
                          前端:react-leaflet 渲染 + 客户端插值 + 区域切换器 + 解说/告警侧栏
```

布局(1920×1080):左侧地图主区,右侧固定宽侧栏(上=解说流,下=告警面板),顶部状态条(数据时延、在视数、退避状态),底部常驻免责条。

## 6. 合规约束(硬性)

1. **密钥隔离**:OpenSky 凭证与 DeepSeek 密钥仅存于后端 `.env`,绝不进前端产物、绝不经 SSE/接口下发;前端不直连任何上游。
2. **单点轮询 + 限流(含多区域/全球护栏)**:后端单实例统一轮询,**任一时刻只有一个活动范围**(预设/视野/全球),切换只改下一周期的 bbox/间隔,**绝不**为多区域并发开多路轮询;令牌刷新、429/5xx 指数退避带抖动。多区域/全球新增护栏:
   - **视野跟随防抖**:前端地图停止拖动/缩放 **≥ 500ms** 才下发新 bbox,避免连发。
   - **面积上限**:`viewport` bbox 经纬跨度超过 `MAX_VIEWPORT_DEG`(默认约 60°)时,自动夹取或改用全球模式,防止超大查询。
   - **全球降频**:全球 `states/all` 数据量大、扣额度高,使用 `GLOBAL_POLL_INTERVAL_MS`(默认 30s)更长间隔;必要时对返回做抽稀/聚合,仅保证演示流畅而非逐架完整。
   - **限流自保**:连续 429 时自动延长间隔并提示;恢复后回落。
3. **AI 客观克制**:解说与告警 system prompt 强制——只做客观、中性描述;对紧急代码**只**说明通常含义与管制通常处置;**禁止**渲染、臆断、预测某架飞机出事/坠毁/伤亡;禁止给操作建议;不确定即说明不确定。
4. **非航行用途声明**:界面常驻「数据可能延迟/不完整,仅供学习,不用于实际航行或调度」;不得呈现为权威/实时可依赖的航行信息源。
5. **数据最小化**:仅内存短时轨迹窗口,不长期落库;不采集用户数据。

## 7. 验收标准(整体)

- [ ] 前端构建产物与所有网络响应中**搜索不到**任何密钥/凭证字符串。
- [ ] 后端轮询稳定在约 10s/次;断网或 429 时进入退避且前端有"数据暂不可用/恢复"状态提示,不崩溃。
- [ ] 地图显示当前聚焦区域的在视飞机,图标按航向旋转、按高度着色;两次快照间飞机平滑滑行无跳变。
- [ ] 切换预设区域 / 跟随视野 / 全球视图后,数据在下一周期随之更新;切换时不在新范围的目标被清理;顶栏显示当前区域/模式。
- [ ] 视野跟随有 ≥500ms 防抖;超大视野被夹取或转全球;全球用更长间隔;切换/拖动不触发 429 刷屏。
- [ ] 每约 30s 解说侧栏新增一段中性文本;文本不含臆断性/事故性表述。
- [ ] 注入或遇到 squawk 7500/7600/7700 时,在 5s 内出现对应告警卡片,文案仅说明通常含义且含"不代表发生事故"类免责。
- [ ] 快速下降(≤ -10 m/s 持续 2 周期)与疑似盘旋(90s 内 ≥270°)能触发告警且带 evidence。
- [ ] 同目标同类告警在冷却期内不重复刷屏。
- [ ] 免责条始终可见。

## 9. 待确认 / 风险

- OpenSky 对某 bbox 的覆盖密度依赖 ADS-B 接收站,夜间或低空目标可能稀少——属数据源限制,符合"数据可能不完整"声明。
- 匿名/未认证 OpenSky 速率更低;需 OAuth2 凭证以获得更高额度(已在约束内)。
- **多区域/全球的额度与数据量**:OpenSky 按查询面积/全量扣额度,频繁切换大范围或全球易触发 429;靠 §6 护栏(防抖、面积上限、全球降频、退避)控制,极端情况下宁可降频/夹取也不并发。
- **全球视图渲染量**:全量可能上万架,前端逐架渲染压力大;必要时按缩放层级抽稀/聚合,优先保证流畅与稳定。
- **活动范围为全局单例**:所有大屏共享同一活动范围,某客户端切换会影响全部连接(符合"单点轮询"取向);如需各端独立范围属更大改动,本版不做。
- `deepseek-v4-flash` 为本项目选定模型 id;若实际 DeepSeek 模型名不同,以后端配置项 `DEEPSEEK_MODEL` 覆盖。
